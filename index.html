The following git commands were explained:

  • git config - sets the author name and email address respectively to be used with commits.
  • git init - start a new repository.
  • git clone - obtain a repository from an existing URL.
  • gi add - adds a file to the staging area.
  • git commit -m - records or snapshots the file permanently in the version history.
  • git status - ists all the files that have to be committed.
  • git -v - check git application version
  • git restore - unstage uncommitted local changes.
  • git push - sends the committed changes to remote repository.
  • git pull - fetches and merges changes on the remote server to working directory.
  • git checkout - switch from one branch to another.
